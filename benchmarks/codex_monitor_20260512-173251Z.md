**Q:** Transformer 的 self-attention 公式是什么？

- top sources: ch4(2).pdf@Page 41/97 (score=0.708) · ch9.pdf@Page 49/70 (score=0.678) · ch4(2).pdf@Page 66/97 (score=0.662)

**Answer:**

```
提供的资料没有给出完整的标准形式 `$QK^\top/\sqrt{d_k}$` 公式；课件中明确展示的是 **Masked Self-Attention 中 attention 分数经过 softmax 得到权重** 的计算方式。

可概括为：

$$
\alpha'_{i,j}=\frac{\exp(\alpha_{i,j})}{\sum_t \exp(\alpha_{i,t})}
$$

其中被 mask 的位置会被设为一个很大的负数，如 `$-100000000$`，因此经过指数运算后近似为 `$0$`，不会被关注到；课件示例中：

$$
\alpha'_{1,1}=\frac{\exp(\alpha_{1,1})}{\exp(\alpha_{1,1})+\exp(\alpha_{1,2})+\exp(\alpha_{1,3})+\exp(\alpha_{1,4})}
$$

并展示了 mask 后后两个位置贡献为 `$0$` 的情况。[Source: ch4(2).pdf, Page 75/97]

Self-attention 是 Transformer 的核心组成，用于捕获数据或特征内部的相关性，并可组成 Multi-head Self-attention。[Source: ch4(2).pdf, Page 41/97]
```

**Answer:**

```
Transformer 使用多个 attention 头，核心目的是让模型从多个表示子空间/角度同时捕获序列内部的相关性；因为 self-attention 本身擅长捕获数据或特征的内部相关性，而 multi-head self-attention 是由多个 self-attention 组成的。 [Source: ch4(2).pdf, Page 41/97]

- 在 BERT 这类 Transformer 编码器中，每个位置的表示会通过自注意力机制充分交互，从而得到更丰富的上下文表示。 [Source: ch9.pdf, Page 49/70]
```

- path: `graphrag` · elapsed: 18.4s · answer_len: 484
- top sources: ch9.pdf@Page 19/70 (score=0.833) · ch1.pdf@Page 30/75 (score=0.682) · ch9.pdf@Page 68/70 (score=0.001)

**Answer:**

```
**数据稀疏问题**：在 $n$-gram 语言模型中，很多高阶 $n$-gram 在训练语料中没有出现或出现次数很少，导致模型无法可靠估计其概率；因此常需要在高阶 $n$-gram 不存在时退回到低阶分布来估计。[Source: ch9.pdf, Page 19/70]

**常用平滑方法**包括：

- **Jelinek-Mercer 平滑**：通过高阶和低阶 $n$-gram 模型的线性插值来融合概率估计。[Source: ch9.pdf, Page 19/70]
- **Witten-Bell 平滑**：属于常见语言模型平滑算法之一。[Source: ch9.pdf, Page 19/70]
- **Kneser-Ney 平滑**：也是常见的 $n$-gram 平滑算法之一。[Source: ch9.pdf, Page 19/70]
- **回退 / 插值思想**：核心是“如果 $n$-gram 存在则使用其本身计数，如果不存在再退后到低阶分布”；有些方法还会对高阶和低阶模型做线性插值。[Source: ch9.pdf, Page 19/70]
```

**Q:** What does the Markov assumption say about state transitions in a sequence?

- path: `graphrag` · elapsed: 10.5s · answer_len: 446
- top sources: ch3.pdf@Page 2/51 (score=0.351) · ch3.pdf@Page 41/51 (score=0.169) · ch3.pdf@Page 17/51 (score=0.132)

**Answer:**

```
The **Markov assumption** says that in a state sequence, the probability of the next state depends only on the current/previous state, not on the entire past history.

In the HMM/Viterbi recurrence, this appears as a transition factor $a_{ji}$ from previous state $j$ to current state $i$, combined with the best probability up to the previous time step $\delta_t(j)$, rather than conditioning on all earlier states. [Source: ch3.pdf, Page 32/51]
```

**Q:** What is the role of positional encoding in the Transformer architecture?

- path: `graphrag` · elapsed: 12.6s · answer_len: 601
- top sources: ch4(2).pdf@Page 66/97 (score=0.531) · ch4(2).pdf@Page 69/97 (score=0.531) · ch4(2).pdf@Page 89/97 (score=0.531)

**Answer:**

```
Positional encoding’s role is to add token position/order information to the embeddings before they enter the Transformer attention layers.

- In the encoder, positional encoding is added to the input embedding before self-attention. [Source: ch4(2).pdf, Page 66/97]
- In the decoder, positional encoding is added to the shifted output embedding before masked self-attention. [Source: ch4(2).pdf, Page 66/97]
- This helps the Transformer use self-attention while still keeping track of sequence order, which is important for sequence tasks such as machine translation. [Source: ch4(2).pdf, Page 41/97]
```
